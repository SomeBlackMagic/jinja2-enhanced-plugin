Version 2.3.0

Assignment Tags, Type System & Navigation Enhancements

🔌 New Extension Point

  • Added assignmentTagProvider — register custom tags that bind variables via as keyword; includes Jinja2AssignmentTagDescriptor, Jinja2AssignmentTagProvider, Jinja2AssignmentTagRegistry, and built-in Jinja2AssignmentTagScopeVariableProvider exposing bound variables in scope for completion and inspections (enables framework integrations such as Ansible)

🔗 References & Find Usages

  • Added Jinja2BlockReference — Ctrl+Click navigation for {% block %} names through template inheritance chains with cycle-safe parent traversal
  • Added autocompletion for inherited block names in child templates
  • Added Find Usages support for variables bound via assignment tags (as keyword) through Jinja2AssignmentTagBindingPsi
  • Added Find Usages and nested attribute resolution for assignment tag variables; restricted search scope to Jinja2 files

💡 Code Completion

  • Enhanced Jinja2CompletionContributor with nested member completion for assignment tag variables (e.g., mysql.host, mysql.port)
  • Added resolveChildBySegment for dotted and bracket-notation key resolution in nested attribute completions

📖 Documentation

  • Added Jinja2AssignmentTagVariableDocumentation — Ctrl+Q hover documentation for assignment tag keywords and their bound variables

⚙️ Parser & Lexer

  • Added tuple unpacking in {% for %} loops — supports parenthesized and plain forms: {% for a, b, c in seq %}, {% for (a, b) in seq %}
  • Added support for post-filter binary operators in inline-if conditions: {% if value|lower in ["a", "b"] %}, {% if value|lower == "x" %}
  • Added nested single-block statements in control-flow bodies — import, from-import, set, include now parse correctly inside {% if %} and {% for %} blocks
  • Added inline assignment tag resolution inside conditional blocks
  • Refactored StatementDispatcher — introduced parseStatementBody() for keyword-first dispatch of single-block statements

🔍 Inspections

  • Improved Jinja2UndefinedAttributeInspection — early exit for provider variables with no known children, eliminating false positives
  • Added Jinja2DoMethodCallInspection — validates schema-backed methods used in {% do %} expressions; detects missing methods, non-callable fields, and improper side-effect-free expressions

🧩 Type System

  • Introduced Jinja2BuiltInTypeRegistry — defines schemas for built-in Python types (dict, list) with method completion support
  • Added Jinja2TypeParam and Jinja2TypeMemberAccessPath for modeling parameterized type fields and static member access paths
  • Added Jinja2LocalTypeResolver for inferring and resolving locally scoped types

⚙️ Configuration

  • Overhauled Jinja2 API configuration: replaced Jinja2DelimiterProvider / Jinja2EnvironmentConfigProvider with Jinja2ApiConfigOverride (patch-style builder) and Jinja2ApiConfigProvider; Jinja2ApiConfigRegistry merges default settings with provider overrides
  • Added Jinja2UndefinedBehavior and Jinja2AutoescapeMode enums; introduced Jinja2SyntaxOptions record for immutable line-prefix syntax configuration
  • Added Jinja2ApiSettingsConfigurable and Jinja2ApiSettingsPanel — Settings UI with validation for delimiters, line prefixes, and extensions

🏗️ Infrastructure

  • Migrated to IntelliJ Platform Plugin 2.x (org.jetbrains.intellij.platform 2.2.0); updated sinceBuild to 241 (IntelliJ 2024.1)
  • Improved variable resolution in multi-host injected contexts via collectVisibleVariablesFromInjectedHostContext in Jinja2ScopeUtil
  • Replaced deprecated IntelliJ Platform API usages across inspections, registries, and notification providers
  • Replaced custom SecondaryJinjaHostLanguage logic with PlainTextLanguage in multi-host injection tests

Version 2.2.0

Extension Points, Framework Support & Advanced Inspections

🔌 New Extension Points

  • Added tagBehaviorProvider — register custom Jinja2 tag behaviors (block/inline types, closing tag aliases); includes DefaultJinja2TagBehaviorProvider with built-in tags (trans, autoescape, break, continue, etc.)
  • Added liveTemplateContextProvider — contribute framework-specific live template context types (e.g., Ansible, Django) without IDE restarts
  • Added delimiterProvider — define custom block, variable, and comment delimiters; integrated into lexer, parser, brace matcher, syntax highlighter, and auto-closing handler
  • Added scopeVariableProvider — contribute context-aware variables (e.g., loop.* in {% for %}, caller/varargs/kwargs in {% macro %}); used in completion and undefined-variable inspection
  • Added macroModuleProvider — declare virtual macro modules for framework-specific or runtime-defined macros; integrated into completion and imported macro/variable resolvers
  • Added injectionLanguageProvider — configure dynamic Jinja2 language injection rules (WHEN_DELIMITERS_PRESENT, ALL_STRING_LITERALS, WHOLE_FILE); supports user-defined injection settings
  • Added inspectionRuleProvider — suppress false positives for undefined variables/attributes and unused constructs via whitelist/blacklist patterns with wildcard support
  • Added elementDocumentationProvider — provide rich HTML documentation for filters, tests, functions, variables, macros, blocks, and tags; activates the documentation popup
  • Added typeSchemaProvider — register named type schemas and bind variable names to schema types; drives attribute completion and Jinja2UndefinedAttributeInspection
  • Added templateRootProvider — declare framework-specific template root directories; used in template path resolution, completion, and reference lookup

🔍 Inspections

  • Added Jinja2UndefinedAttributeInspection — warns on unresolved nested attribute or index access on statically known paths; respects guards (is defined, default filter, if conditions) and ignore annotations
  • Improved Jinja2UndefinedVariableInspection — delegates nested attribute validation to Jinja2UndefinedAttributeInspection, respects scope variable providers and inspection rule providers
  • Improved Jinja2InvalidTagUsageInspection — replaced boolean flags with depth counters for accurate validation inside nested call, macro, and trans blocks
  • Improved Jinja2UnsupportedStatementInspection — added StatementCapabilityPolicy abstraction for loop control extensions; skips inspection inside {% raw %} blocks
  • Improved Jinja2LogicalErrorInspection — extended token recognition and fixed validation of IS NOT constructs
  • Improved Jinja2UnusedConstructsInspection — replaced substring checks with boundary-aware identifier matching utilities
  • Improved Jinja2InvalidInheritanceInspection — detects non-whitespace content before {% extends %}
  • Refactored Jinja2DuplicateMacroInspection — switched to PsiElementVisitor with visited-set deduplication

🔗 References & Find Usages

  • Added Jinja2FindUsagesHandlerFactory for project-scoped Find Usages on variables, macros, and block definitions
  • Added Jinja2MacroParameterPsi implementing PsiNamedElement — enables Find Usages on macro parameters with correct scope isolation
  • Added Jinja2ImportedMacroResolver and Jinja2ImportedVariableResolver for cross-file macro and variable resolution
  • Added Jinja2ModuleAliasAttributeReference for attribute-based module alias lookups
  • Added Jinja2TemplateFileResolver — multi-step path lookup with filename-index fallback for template resolution

💡 Code Completion

  • Added Jinja2LocalDictSchema — resolves inline {% set %} dictionary assignments for attribute and bracket-notation completions
  • Enhanced completion to include scope variables, type schema fields, and template-root-relative path suggestions

⚙️ Parser & Lexer

  • Added UnknownTagBlockParser with TagRegistry / DefaultTagRegistry — parses unknown or custom block tags with graceful recovery
  • Added Jinja2DelimiterTransformingLexer for lossless token transformation based on custom delimiters
  • Added Jinja2MacroParameterPsi mapping in Jinja2ParserDefinition

🧪 Testing & Reliability

  • Added ExpressionAdvancedTest and UnknownStatementBodyTest for complex expression and custom-tag lexer scenarios
  • Added tests for all new extension points, registries, resolvers, and inspections
  • Improved ignore-comment handling with CachedValuesManager-backed rule retrieval and boundary-aware extraction utilities

Version 2.1.0

Code Intelligence, Editor Productivity & Refactoring Foundations

🧠 Code Completion

  • New Jinja2CompletionContributor providing context-aware completions for tags, filters, tests, and variables
  • Smart completion context resolution via Jinja2CompletionContextResolver — correctly distinguishes TAG_KEYWORD, FILTER, TEST, VARIABLE, and UNKNOWN contexts
  • Nested variable member completions with support for both dot (foo.bar) and bracket (foo["bar"]) notation, including multi-level nesting
  • Built-in variable, filter, and test descriptors provided by DefaultJinja2VariableProvider, DefaultJinja2FilterProvider, and DefaultJinja2TestProvider
  • Built-in global function descriptors via DefaultJinja2FunctionProvider
  • New variableProvider extension point — third-party plugins can now contribute custom variable, filter, and test completions
  • Lookup items with type annotations and descriptions via Jinja2LookupElementFactory

⌨️ Editor Typing Experience

  • New Jinja2TypedHandler — automatically inserts closing delimiters (%}, }}, #}) when typing their opening pairs
  • Smart handling of already-closed delimiters to avoid duplication
  • Correct behavior for unmatched or partial opening braces

🎨 Syntax & Rainbow Highlighting

  • Rainbow block delimiter highlighting via Jinja2BlockRainbowAnnotator and Jinja2BracketsRainbowAnnotator
  • Expression-level rainbow highlighting via Jinja2ExpressionsRainbowAnnotator
  • New rainbow level color keys (RAINBOW_L0–RAINBOW_L4) in Jinja2SyntaxHighlighter, configurable in the Color Settings page
  • Updated Default and Darcula color schemes with rainbow-level styling
  • New Jinja2SemanticHighlightingAnnotator for contextual semantic coloring

🧩 Code Navigation

  • New Jinja2CodeBlockSupportHandler enables code block navigation and marker highlighting for {% for %}…{% endfor %}, {% if %}…{% endif %}, and other paired constructs
  • Accurate range calculation for nested blocks and incomplete block pairs

✏️ Commenting

  • New Jinja2Commenter with full line and block comment support using Jinja2's {# … #} syntax
  • Toggle line/block comment works correctly with multi-line selections and nested constructs

🔤 Spellchecking

  • New Jinja2SpellcheckingStrategy integrates IDE spellchecking into Jinja2 template files

🗺️ Breadcrumbs

  • New Jinja2BreadcrumbsProvider adds breadcrumb navigation in the editor gutter for nested template structures

🎛️ Code Style & Formatting

  • Added Jinja2ColorSettingsPage for full syntax color customization
  • New Jinja2FormattingModelBuilder, Jinja2LanguageCodeStyleSettingsProvider, and Jinja2CodeStyleSettingsProvider for configurable code formatting
  • Improved block and macro body alignment in the formatter

🧪 Testing & Reliability

  • Extensive new test coverage for all v2.1 features: completion, context resolution, typed handler, rainbow annotators, code block handler, commenter, breadcrumbs, PSI helpers, and descriptors
  • New Jinja2MacroDefinitionPsi#getParameterNames() and Jinja2SetStatementPsi#getAssignedVariableNames() helper methods with full test coverage
  • Jinja2ElementFactory added to simplify PSI element creation in tests and runtime code
