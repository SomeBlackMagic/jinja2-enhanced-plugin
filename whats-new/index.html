Version 2.0.0-rc1

Complete Plugin Rewrite — Enhanced Parsing & Editor Features

🎯 Core Parser Improvements

  • Completely redesigned parser architecture with hand-written custom parser for better accuracy and maintainability
  • Modular expression and statement parsing with specialized parsers for each construct
  • Enhanced support for all Jinja2 operators including is tests, power operator, comparison chains
  • Improved handling of inline-if (ternary) expressions and complex filter chains
  • Support for if conditions in for loops
  • Better parsing of macros with keyword arguments, defaults, varargs, and kwargs
  • Enhanced import/from statement parsing with proper alias handling
  • Nested raw block support with depth tracking

🎨 Editor & UI Enhancements

  • Rainbow bracket highlighting for better code readability with separate annotators for blocks and brackets
  • Improved semantic highlighting with contextual syntax coloring
  • Enhanced brace matching support for Jinja2 delimiters
  • Improved code formatting with spacing builder and customizable code style settings
  • Built-in documentation viewer with embedded tool window and editor tabs

🔍 Code Quality & Inspections

  • New template inspections for detecting common errors and issues
  • Better error reporting with intelligent filtering
  • Enhanced PSI structure for more reliable code analysis

🧪 Testing & Reliability

  • Comprehensive lexer test coverage (100+ test cases) for all token types and edge cases
  • Extensive parser test suite covering all Jinja2 constructs and syntax variations
  • Tests for complex nested structures, whitespace handling, and error conditions
  • Improved stability and reduced false positives

📚 Documentation

  • Comprehensive Jinja2 syntax analysis documentation
  • Technical specification and architecture guides
  • In-IDE documentation access for quick reference

⚡ Performance & Infrastructure

  • Optimized PSI utilities and file view provider
  • Streamlined error handling without external dependencies
  • Better handling of unterminated blocks and malformed templates
  • Support for whitespace trim modifiers (-, +) throughout

Note: This is a major rewrite focused on establishing a solid foundation. Some features from 1.x may be reimplemented differently or scheduled for future releases.
